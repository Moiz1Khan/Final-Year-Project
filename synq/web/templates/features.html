Product

How Synq works

Four steps: speak → understand → act → respond. Your session stays tied to this browser after login; the voice client uses the same account via data/auth_token.json.

  1. Speak — Wake word, face gate, or open listening (see your config.yaml).
  2. Understand — Cloud or local pipeline maps speech to intents and skills.
  3. Act — Productivity, Gmail, Calendar, desktop routines, and more.
  4. Respond — Spoken replies; this console shows stats and history.

Calendar

Meetings

Schedule events, request Meet links, and see upcoming items on the overview when Google is connected.

Try: “Schedule a meeting tomorrow at 3pm.”

Gmail

Mail

List unread, hear summaries, and send mail with your connected Google account.

Try: “What are my unread emails?”

Tasks

Tasks & reminders

Tasks live in your Synq database; open counts appear on the overview.

Try: “Add a task to finish the report Friday.”

Desktop

PC control

Open apps, sites, search, media keys, and routines — restricted by allowlists in config.yaml.

Try: “Open Chrome” or “Search the web for Python asyncio.”

Voice

Agent

Run python scripts/run_voice_agent.py (or main.py) on the machine with the mic. Use voice.backend: synq_cloud to pair with this server.

Context

Memory

Conversation turns are stored for context and show up under Activity. Optional context monitoring can log foreground app usage separately.
